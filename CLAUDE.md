## Features

### 1. Web Scraping with Stealth Playwright

A fully headless browser scraper built on Playwright's async Chromium engine. Accepts any public URL and returns
structured data: full HTML, plain text content, extracted main article body (via Mozilla's Readability algorithm),
Open Graph / meta tag metadata, final resolved URL after redirects, and optionally a full-page base64 PNG screenshot.

Anti-detection is handled by `playwright-stealth`, which injects init scripts before navigation to patch all
JavaScript automation signals that Cloudflare and similar WAFs fingerprint — `navigator.webdriver`, plugin lists,
`navigator.userAgentData`, WebGL vendor strings, and ~15 other properties. Without this, headless Chromium is
reliably blocked by Cloudflare-protected sites.

Concurrency is capped at 3 simultaneous Chromium instances via an `asyncio.Semaphore` to prevent resource
exhaustion under burst traffic. Each request creates and fully tears down its own browser → context → page chain.
Navigation uses a two-phase strategy: wait for the `load` event first (reliable), then attempt `networkidle`
with a 5-second grace period to let late-loading JS settle without hard-failing on sites with persistent
third-party network activity. An optional `waitForSelector` parameter allows callers to wait for a specific
DOM element before extraction.

Main files: @app/routers/scrape.py

### 2. YouTube Transcript Fetching

Two endpoints for retrieving YouTube video data without using the official YouTube Data API or API key.
Uses the `youtube-transcript-api` library routed through the configured proxy to avoid YouTube IP bans.

- `GET /yt?videoId=&language=` — fetches the full transcript for a specific video in the requested language,
  returned as a list of timed text segments (`text`, `start`, `duration`).
- `GET /yt-list?videoId=` — lists all available transcript tracks for a video, including language code,
  whether the track is auto-generated by YouTube, and whether it can be translated.

Both endpoints share the same proxy selection logic (`isFree` flag), error classification (connection errors
return 503, YouTube IP bans return 429 with a hint to rotate the proxy identity), and client factory
`get_youtube_api_client`. When an IP ban is hit, the error message links the caller directly to the
`POST /proxy/new-identity` endpoint to get a new exit IP.

Main files: @app/routers/youtube.py

### 3. Statistics Ingestion into ClickHouse

A single `POST /stats/` endpoint that bulk-inserts vocabulary learning stat records into a ClickHouse table.
The request body carries the target table name and a list of stat items, each representing one word's spaced
repetition state: language pair, word ID, external ID, SM-2 interval and repetition count, last result,
and timestamps for when the record was added, last updated, and when the next review is due.

Column ordering for the ClickHouse insert is derived dynamically from the union of all keys present in the
payload, so the endpoint is tolerant of sparse records where some fields may be absent.

Main files: @app/routers/stats.py, @app/database.py

### 4. Proxy Infrastructure

A unified proxy layer that all features (scraper, YouTube, proxy test) share. Supports two modes selected
by an `isFree` boolean flag present on every endpoint:

- **Paid residential proxies** — five providers are supported: BrightData, Oxylabs, SmartProxy, IPRoyal,
  FloppyData. The active provider is selected via the `PROXY_PROVIDER` env var. Each provider has its own
  session ID username format (e.g. FloppyData: `user-{u}-type-residential-session-{id}-country-{cc}-rotation-{min}`).
  A global session ID (`_current_session_id`) keeps the same exit IP across requests until rotated.
- **Tor** — SOCKS5 proxy via a containerized Tor instance (currently commented out in docker-compose).

Two management endpoints live under `/proxy`:

- `GET /proxy/test` — hits `httpbin.org/ip` twice (direct and through proxy) and returns both IPs plus a
  `proxy_working` boolean to confirm the proxy is actually masking the server's real address.
- `POST /proxy/new-identity` — for paid proxies: generates a new UUID session ID so the next request uses
  a different residential IP (instant). For Tor: sends `SIGNAL NEWNYM` to the Tor control port (port 9051)
  to request a new circuit, then verifies the new exit IP.

For Playwright specifically, `get_playwright_proxy()` formats credentials as separate `username`/`password`
fields because Playwright ignores credentials embedded in the server URL.

Main files: @app/proxy.py, @app/routers/tor.py, @app/config.py
